<p>To launch a cluster you must first generate authentication keys. Next you must
configure the parameters of your cluster. Finally you can invoke Terraform to
create the cluster.</p>

<li><a href="https://www.terraform.com" rel="nofollow">Terraform</a></li>

<h2>
<a aria-hidden="true" class="anchor" href="#for-arch-linux" id="user-content-for-arch-linux"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>For Arch Linux</h2>
<p>Arch Linux has packages for both Terraform and cfssl. Install the <code>terraform</code>
package from the official repository. And the <code>cfssl</code> package from the
community repository.</p>
<h1>
<a aria-hidden="true" class="anchor" href="#keys" id="user-content-keys"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>Keys</h1>
<p>First create a set of SSH keys. These keys will be used to authenticate with our
Kuberentes cluster.</p>
<p>This command will generate a new set of SSH keys, and place them in the <code>keys</code>
directory:</p>
<pre><code>$ ssh-keygen -t rsa -b 4096 -f keys/k8s_id_rsa
</code></pre>
<p>Be sure not to set a password on these keys.</p>
<p>Next add this SSH key to DigitalOcean on this <a href="https://cloud.digitalocean.com/settings/security" rel="nofollow">settings page</a>.</p>
<p>Finally add this key to your ssh agent:</p>
<pre><code>$ ssh-add ./keys/k8s_id_rsa
</code></pre>

<h1>
<a aria-hidden="true" class="anchor" href="#provision" id="user-content-provision"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>Provision</h1>
<p>To provision the Kubernetes cluster with the commands:</p>
<pre><code>$ terraform init k8s-do
$ terraform plan -out plan.tfplan k8s-do
$ terraform apply plan.tfplan 
</code></pre>
<p>After the process succeeds copy the <code>k8s-do/secrets/setup_kubectl.sh</code> file to
the <code>secrets</code> directory with:</p>
<pre><code>$ cp k8s-do/secrets/setup_kubectl.sh secrets
</code></pre>
<p>This file contains the information needed to authenticate with the new cluster.</p>
