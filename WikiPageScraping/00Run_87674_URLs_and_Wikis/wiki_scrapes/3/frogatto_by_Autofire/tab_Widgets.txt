          <p>Widgets are components of Frogatto's built-in GUI capabilities. They behave much like regular objects in FFL, but with a limited number of events. To use a widget, you must attach it to an object via set_widgets() and they can be cleared from an object with clear_widgets(). (See [Object-Functions](Object Functions) for the complete set.) All widgets can be scripted without the use of widgets if different behavior is needed. For a live example of how to use widgets, you could have a look at ~/modules/frogatto/data/objects/experimental/button_controller.cfg. The FSON that describes the widget acts like a template to create the widget from.</p>

<p>Common to all widgets.</p>
<h3>
<a aria-hidden="true" class="anchor" href="#variables" id="user-content-variables"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>Variables</h3>
<ul>
<li>type: A string, like "button" or "text_editor". See the widget types list for details. This field is required so that the factory function that builds widgets knows what sort of widget to create.</li>
<li>id: A string that can be used to identify the widget</li>
<li>x/y/width/height/rect/draw_area: Different ways of specifying the position and area of the widget.</li>
<li>on_process: FFL formula that is performed every cycle.</li>
<li>tooltip: A string containing the tooltip to be displayed when the mouse is hovering over the widget.</li>
<li>visible: A boolean indicating whether the widget is visible or not.</li>
<li>enabled: A boolean indicating whether the widget process events.</li>
</ul>
<h3>
<a aria-hidden="true" class="anchor" href="#variables-that-should-be-avoided-when-possible" id="user-content-variables-that-should-be-avoided-when-possible"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>Variables that should be avoided when possible</h3>
<ul>
<li>zorder: Specifies the z-ordering to use when the widget is grouped with other widgets.</li>
<li>align_h: Horizontal alignment. One of "top", "center", or "bottom".</li>
<li>align_v: Vertical alignment. One of "top", "center", or "bottom".</li>
</ul>

<p>A button you can click on.</p>
<h3>
<a aria-hidden="true" class="anchor" href="#variables-1" id="user-content-variables-1"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>Variables</h3>
<ul>
<li>label: A widget specifying the face text of the button. This can be <em>any</em> other widget. To be consist with frogatto style of dialogs the graphical_font widget should be used.</li>
<li>padding: A list containing x/y padding values. For example, [12,6] would give us 12 pixels of horizontal padding between the text and the outer edge of the button, and only half that vertically. (This could also be given by the variables hpad and vpad, but it's neater to do it with padding.)</li>
<li>resolution: Either "normal" or "double". Makes the border of the button either single or double sized.</li>
<li>style: Either "default" or "normal". Difference unknown.</li>
</ul>

<li>width: How wide the text-box is.</li>
<li>font_size: How many points should the font be?</li>
<li>color: What color should the text be?</li>

<h3>
<a aria-hidden="true" class="anchor" href="#events-1" id="user-content-events-1"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>Events</h3>
<ul>
<li>on_change: Fired when the text is edited.</li>
<li>on_move_cursor: The cursor is moved.</li>
<li>on_enter: The return key was pressed.</li>
<li>on_tab: Tab-completion should run. (Tab was pressed.)</li>
<li>on_escape: The escape key was pressed.</li>
<li>on_begin_enter: Unknown.</li>
<li>on_change_focus: Focus is lost or gained.</li>
</ul>
