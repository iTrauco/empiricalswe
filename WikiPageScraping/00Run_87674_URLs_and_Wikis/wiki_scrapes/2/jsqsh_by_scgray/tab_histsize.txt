<p><code>histsize</code> - Controls the number of SQL statements retained in history</p>
<h2>
<a aria-hidden="true" class="anchor" href="#description" id="user-content-description"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>Description</h2>
<p>The <code>${histsize}</code> variable determines the number of SQL statements
that is retained historically (the default is 50). Each time a
SQL statement is executed with the <code>\go</code> command, that statement
is saved away and be re-edited and re-executed later if desired.</p>
<p>The <code>\history</code> command shows the last <code>${histsize}</code> SQL statements
executed and <code>\buf-edit</code> can be used to edit and re-run a
previously executed statement. Also, see <strong><a class="internal present" href="/scgray/jsqsh/wiki/buffers">buffers</a></strong> for
details on how to refer to specific buffers.</p>
