<h2>
<a aria-hidden="true" class="anchor" href="#description" id="user-content-description"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>Description</h2>
<p>Displays a list of drivers that are currently registered with
jsqsh. An asterisk (*) will be displayed next to drivers for
which the actual JDBC implementation (.jar file) is available.</p>
<p>JSqsh does not come bundled with any JDBC drivers therfore unless
you already have a driver available in your classpath (via the
$CLASSPATH environment variable), you will likely see a list of
drivers without any marked as available:</p>
<pre><code>1&gt; \drivers
+--------------------------------------------+-------------+----------
| Target                                     | Name        | URL      
+--------------------------------------------+-------------+----------
|   Starschema BigQuery                      | bigquery    | jdbc:BQDr
|   IBM Data Server (DB2, Informix, Big SQL) | db2         | jdbc:db2:
|   IBM DB2 z/OS                             | db2zos      | jdbc:db2:
|   Apache Derby Server                      | derby       | jdbc:derb
|   Apache Derby Embedded                    | derbyembed  | jdbc:derb
...
</code></pre>
<p>(note that the output shown above is truncated to fit this help
text nicely).</p>
<p>Any one of the following mechanisms may be used to make a driver available
to jsqsh:</p>
<ol>
<li>
<p>Each driver may individually have it's classpath set via the <a class="internal present" href="/scgray/jsqsh/wiki/setup">\setup</a>
command (run <code>\setup</code> and navigate to the driver you for wich you wish
to provide a classpath).  This is the preferred method for making a driver
available (see <a class="internal present" href="/scgray/jsqsh/wiki/Getting-Started#driver-setup">Driver setup</a> for details).</p>
</li>
<li>
<p>Setting the ${<a class="internal present" href="/scgray/jsqsh/wiki/classpath">classpath</a>} jsqsh variable to a location containing
JDBC jar(s) for the driver(s) you want.</p>
</li>
<li>
<p>Making sure the driver is contained in your CLASSPATH environment
variable, e.g., for UNIX or cygwin:</p>
<pre><code> $ CLASSPATH=/path/to/some/jdbc/driver.jar:"$CLASSPATH"
 $ export CLASSPATH
</code></pre>
<p>or, for Windows:</p>
<pre><code> C:\&gt; set CLASSPATH=/path/to/some/jdbc/driver.jar; %CLASSPATH%
</code></pre>
</li>
<li>
<p>Alternatively, you can add a JDBC driver implementation to jsqsh's
startup jar directory. The path to this directory can vary from
platform to platform but can typically be found in one of the
following locations relative to where the jsqsh binary is located:</p>
<pre><code> ../lib
 ../share
 ../share/jsqsh
</code></pre>
</li>
</ol>

<p>JSqsh comes with a bunch of pre-defined driver definitions built-in,
however you can provide your own additional definitions if you wish
using the jsqsh setup wizard (run <code>\setup</code>). The driver definitions
that are created or edited are located in <code>$HOME/.jsqsh/drivers.xml</code>.</p>
