<h2>
<a aria-hidden="true" class="anchor" href="#description" id="user-content-description"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>Description</h2>
<p>The <code>\select</code> command generates a SELECT statement based upon the
set of tables provided as an argument.</p>
<p>If more than one table is provided to <code>\select</code>, then an appropriate
WHERE clause will attempt to be generated to join the tables together.
If the <code>-n</code> flag is provided, then the WHERE clause will be the "natural
join" of the tables--that is, all columns of the same name and type
will be joined together.  If the <code>-n</code> flag is not provided, then the join
will be created by analyzing the foreign key/primary key relationships
between the tables.</p>
<p>The <code>\select</code> command requires proper meta-data support from the underlying
JDBC driver and may not work properly for all driver implementations.</p>

<a aria-hidden="true" class="anchor" href="#--natural-join--n" id="user-content---natural-join--n"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>--natural-join (-n)</h3>
<p>Cause the join clause that is produced to be based upon the "natural join"
of the columns in the tables. The natural join attempts to join all columns
of the same name and datatype.</p>

placing it into the SQL buffer for execution.</p>
