<h2>
<a aria-hidden="true" class="anchor" href="#synopsis" id="user-content-synopsis"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>Synopsis</h2>

<h2>
<a aria-hidden="true" class="anchor" href="#description" id="user-content-description"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>Description</h2>
<p>The <code>\connect</code> command attempts to establish a JDBC connection to a database
server.  If a connection is successfully established then the connection
becomes the active connection for the current session. If the session already
had a connection established, then the existing connection is closed and
replaced with the new connection.</p>
<h2>
<a aria-hidden="true" class="anchor" href="#connection-options" id="user-content-connection-options"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>Connection options</h2>
<p>The following options may be used to specify information required to establish
a database connection (or whatever your chosen JDBC driver considers a
connection).</p>
<p>In the help description below, you will see references to variables like
<code>${db}</code> or <code>${server}</code>. When an option says that it sets a given variable,
what is referring to is that if you look at a driver definition using the
command <a class="internal present" href="/scgray/jsqsh/wiki/drivers">\drivers</a>, like so:</p>
<pre><code>1&gt; \drivers
+-----------------+-------------+--------------------------------
| Target          | Name        | URL                            
+-----------------+-------------+--------------------------------
| * MySQL         | mysql       | jdbc:mysql://${server}:${port} 
</code></pre>
<p>The option may set one of these variables. Not all variables are meaningful
or used by all drivers.</p>
<h3>
<a aria-hidden="true" class="anchor" href="#--setup--e" id="user-content---setup--e"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>--setup (-e)</h3>
<p>Enters the connection setup wizard. This allow you to easily define named
connections, so that only the connection name needs to be provided to
establish a connection.</p>
<h3>
<a aria-hidden="true" class="anchor" href="#--driverdriver--d-driver" id="user-content---driverdriver--d-driver"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>--driver=<em>driver</em> (-d <em>driver</em>)</h3>
<p>The name of the jsqsh driver definition that will be used for the
connection (use <code>drivers</code> to see a list).</p>
<h3>
<a aria-hidden="true" class="anchor" href="#--serverserver--s-server" id="user-content---serverserver--s-server"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>--server=<em>server</em> (-S <em>server</em>)</h3>
<p>Sets the value of <code>${server}</code></p>
<h3>
<a aria-hidden="true" class="anchor" href="#--portport--p-port" id="user-content---portport--p-port"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>--port=<em>port</em> (-p <em>port</em>)</h3>
<p>Sets the <code>${port}</code> property. This typically defines the listening port
of the server you wish to connect to. Most drivers will provide a
default value for this property if it is not supplied.</p>
<h3>
<a aria-hidden="true" class="anchor" href="#--databasedb--d-db" id="user-content---databasedb--d-db"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>--database=<em>db</em> (-D <em>db</em>)</h3>
<p>Sets the <code>${db}</code> property. This will cause the database context to be set
to the provided database following connection to the server (in the JDBC
world the database context is known as the "catalog"). Not all drivers
support this behavior.</p>
<h3>
<a aria-hidden="true" class="anchor" href="#--usernameuser--u-user" id="user-content---usernameuser--u-user"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>--username=<em>user</em> (-U <em>user</em>)</h3>
<p>Sets the <code>${user}</code> property that determines the username that will be
utilized to connect to the database.</p>
<h3>
<a aria-hidden="true" class="anchor" href="#--passwordpass--p-pass" id="user-content---passwordpass--p-pass"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>--password=<em>pass</em> (-P <em>pass</em>)</h3>
<p>Sets the ${password} property that will be used to authenticate with the
server.</p>
<h3>
<a aria-hidden="true" class="anchor" href="#--jdbc-urlurl--u-url" id="user-content---jdbc-urlurl--u-url"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>--jdbc-url=<em>url</em> (-u <em>url</em>)</h3>
<p>This option allows you to provide an explicit JDBC URL to utilize when
establishing the connection rather than using jsqsh's driver mechanism.
Typically use of this parameter also requires the use of the <code>--jdbc-class</code>
argument as well.</p>
<h3>
<a aria-hidden="true" class="anchor" href="#--jdbc-classclass--c-class" id="user-content---jdbc-classclass--c-class"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>--jdbc-class=<em>class</em> (-c <em>class</em>)</h3>
<p>When utilizing a direct JDBC url (and not a jsqsh driver definition), this
should be utilized to define which JDBC driver class will be utilized for
the connection.</p>
<h3>
<a aria-hidden="true" class="anchor" href="#--domaindomain--w-domain" id="user-content---domaindomain--w-domain"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>--domain=<em>domain</em> (-w <em>domain</em>)</h3>
<p>For Windows based authentication, sets the <code>${domain}</code> property, specifying
the windows domain to be utilized when establishing the connection. Not
all JDBC drivers support this functionality.</p>
<h3>
<a aria-hidden="true" class="anchor" href="#--propnamevalue--o-namevalue" id="user-content---propnamevalue--o-namevalue"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>--prop=<em>name=value</em> (-O <em>name=value</em>)</h3>
<p>This option allows JDBC driver properties to be explicitly set prior to
connecting, and may be listed more than once. For example:</p>
<pre><code>\connect --prop compress=true --prop failover=false
</code></pre>
<p>The set of properties that are available are specific to a given JDBC
driver, so you'll need to consult the documentation for your driver for
details.</p>
<h3>
<a aria-hidden="true" class="anchor" href="#--url-varnamevalue--v-namevalue" id="user-content---url-varnamevalue--v-namevalue"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>--url-var=<em>name=value</em> (-V <em>name=value</em>)</h3>
<p>Allows a JDBC URL variable to be set by name. For certain variables explicit
command line options are available to set a variable, for example ${db} can
be set with the <code>--database</code> (<code>-D</code>) option, other variables may no available
command line option to represent the variable and this option may be used
to set it.  For example, given a driver called <code>mydriver</code> with the following
URL:</p>
<pre><code>jdbc:mydriver:${db};sessionTimeout=$(timeout}
</code></pre>
<p>you could connect using:</p>
<pre><code>\connect --driver mydriver --database mydb --url-var timeout=10
</code></pre>
<p>or even:</p>
<pre><code>\connect --driver mydriver --url-var db=mydb --url-var timeout=10
</code></pre>
<p>or, if you prefer the short form:</p>
<pre><code>\connect -d mydriver -V db=mydb -V timeout=10
</code></pre>
<h3>
<a aria-hidden="true" class="anchor" href="#--new-session--n" id="user-content---new-session--n"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>--new-session (-n)</h3>
<p>Create a new session for this connect without closing the connection
in the current session. You can manage, view, and switch active sessions
with the <a class="internal present" href="/scgray/jsqsh/wiki/session">\session</a> command.</p>
<h3>
<a aria-hidden="true" class="anchor" href="#connection-name" id="user-content-connection-name"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>connection-name</h3>
<p>If provided, this indicates the logical connection name that should be
used to establish the connection. If additional arguments are provided,
then they will override the same options associated with the logical
connection name. See <strong>Named connections</strong> below for details.</p>
<h2>
<a aria-hidden="true" class="anchor" href="#named-connection-options" id="user-content-named-connection-options"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>Named connection options</h2>
<p>Because many options may be required to establish a connection, it can be
quite tedious to provide these options each and every time you connect,
so jsqsh provides the ability to establish a logical name for a given set
of connection properties, and then use this name instead of providing
all of the connection properties. Such named connections can be created
and managed via the Connection Setup Wizard (using the <a class="internal present" href="/scgray/jsqsh/wiki/setup">\setup</a> command)
or more manually via the following options.</p>
<p>The <strong>NAMED CONNECTIONS</strong> section, below, provides examples of using these
options.</p>
<h3>
<a aria-hidden="true" class="anchor" href="#--list--l" id="user-content---list--l"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>--list (-l)</h3>
<p>Lists all available logical connection names. See <strong>Named connections</strong>
below for details.</p>
<h3>
<a aria-hidden="true" class="anchor" href="#--addname--a-name" id="user-content---addname--a-name"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>--add=<em>name</em> (-a <em>name</em>)</h3>
<p>This option is used to define (or replace) a logical connection name.

<h3>
<a aria-hidden="true" class="anchor" href="#--update--r" id="user-content---update--r"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>--update (-R)</h3>
<p>This option is used to merge together the settings in an existing name
connection with any additional options you provide. For example, if your
password was changed on a given server you can do:</p>
<pre><code>\connect -Pnewpass --update proddb
</code></pre>
<p>where <em>proddb</em> is the named connection you wish to update.</p>

<h3>
<a aria-hidden="true" class="anchor" href="#--show-password--x" id="user-content---show-password--x"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>--show-password (-x)</h3>

<h2>
<a aria-hidden="true" class="anchor" href="#named-connections" id="user-content-named-connections"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>Named connections</h2>
<p>Because a relatively significant number of parameters are required to
be passed to \connect in order to establish a connection, jsqsh provides
a relatively simple mechanism by which a "name" can be associated with
the set of parameters required to create the connection.  Later attempts
to connect to the same server can be performed simply by providing the
logical name rather than the full parameter set.</p>
<p>For example:</p>
<pre><code>1&gt; \connect -Usa -Ppassword -Sprod -p5000 -dsybase --add=PROD
</code></pre>
<p>will create a connection to a Sybase server (on host prod, port 5000 as sa)
and associate the name 'PROD' with this set of parameters.</p>
<p>Later attempts to connect to the same server can be accomplished simply
by providing same name back to \connect:</p>
<pre><code>1&gt; \connect PROD
</code></pre>
<p>Any of the parameters that were associated with the logical connection
may be overridden using normal command line parameters. For example:</p>
<pre><code>1&gt; \connect -Usgray PROD
</code></pre>
<p>would cause the connection to be established as 'sgray' (rather than 'sa'
which was original defined).</p>
<p>And similarly, persistent edits can be made to an existing named connection
by re-adding the definition along with the change, like so:</p>
<pre><code>1&gt; \connect --add=PROD -Usgray PROD
</code></pre>
<p>and in the same fashion, a copy with edits can be made:</p>
<pre><code>1&gt; \connect --add=PROD2 -Usgray PROD
</code></pre>
<p>To view the set of logical connection names currently being managed, you
an use the <code>--list</code> argument:</p>
<pre><code>1&gt; \connect --list
+------+--------+--------+------+----------+----------+--------
| Name | Driver | Server | Port | Username | Password | ...
+------+--------+--------+------+----------+----------+--------
| PROD | sybase | prod   | 5000 | sa       | *******  | ...
+------+--------+--------+------+----------+----------+--------
</code></pre>
<p>To delete a logical connection name, use the --remove argument:</p>
<pre><code>1&gt; \connect --remove PROD
</code></pre>
<p>Logical connection names are persistent (they exist between executions
of jsqsh) and are defined in the file <code>$HOME/.jsqsh/connections.xml</code>.</p>
<h2>
<a aria-hidden="true" class="anchor" href="#default-values" id="user-content-default-values"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>Default values</h2>
<p>For any of the variables set by options described in the OPTIONS
section, above, you may provide a default for that variable by
prefixing the variable name with "dflt_" in your session. For example:</p>
<pre><code>\set dflt_port=5555
</code></pre>
<p>would cause default value of <code>${port}</code> to be 5555 unless the <code>-p</code> option
is provided to override that value.</p>
<h2>
<a aria-hidden="true" class="anchor" href="#variables-set" id="user-content-variables-set"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>Variables set</h2>
<p>After a connection is successfully established, the variables described
in the OPTIONS section will be available privately to your session.
Thus, the variable $port would expand to be the port that was used
when connecting to your session.</p>
