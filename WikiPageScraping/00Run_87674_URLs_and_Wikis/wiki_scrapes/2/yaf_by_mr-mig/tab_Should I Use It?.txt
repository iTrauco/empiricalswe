          <p>You should consider giving it a try if you are building an "evolvable" web app and have some room for experiments.</p>
<p>Here is a checklist to help you decide:</p>
<ul class="contains-task-list">
<li class="task-list-item">
<input class="task-list-item-checkbox" disabled="" id="" type="checkbox"/> The app has <strong>non-trivial</strong> functionality (not a form-based CRUD).</li>
<li class="task-list-item">
<input class="task-list-item-checkbox" disabled="" id="" type="checkbox"/> The app <strong>will eventually grow</strong> and become more complex.</li>
<li class="task-list-item">
<input class="task-list-item-checkbox" disabled="" id="" type="checkbox"/> The app will be developed <strong>not only by you</strong>.</li>
<li class="task-list-item">
<input class="task-list-item-checkbox" disabled="" id="" type="checkbox"/> The feature set and requirements will <strong>constantly change</strong>.</li>
<li class="task-list-item">
<input class="task-list-item-checkbox" disabled="" id="" type="checkbox"/> You need to control the quality, retaining the constant development speed.</li>
<li class="task-list-item">
<input class="task-list-item-checkbox" disabled="" id="" type="checkbox"/> You will have some time to "reshuffle" the components once in a while.</li>
<li class="task-list-item">
<input class="task-list-item-checkbox" disabled="" id="" type="checkbox"/> There is a possibility to have the <strong>same type of the project</strong> with the same styles and behaviour in the future.</li>
</ul>
<p>Have you checked 5 of those ticks?
If yes, then we are on a single track.</p>
