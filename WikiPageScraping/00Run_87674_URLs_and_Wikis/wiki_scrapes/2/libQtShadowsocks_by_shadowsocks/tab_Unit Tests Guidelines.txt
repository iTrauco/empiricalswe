          <p>##Naming Convention</p>
<ul>
<li>The unit test class name has an extra <code>_T</code> appendix, i.e. <code>Address_T</code> is the test class for <code>Address</code>.</li>
<li>The file name of header and source are ended with <code>.t.h</code> and <code>.t.cpp</code> respectively.</li>
<li>Avoid ambiguous or vague function names like "test1", "testFunc".</li>
</ul>
