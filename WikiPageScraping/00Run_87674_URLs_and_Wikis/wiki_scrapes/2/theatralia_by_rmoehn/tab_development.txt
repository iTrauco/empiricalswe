<ul>
<li>aim for high degree of distribution from the start</li>
<li>(keep in mind that every user is only one node and later on must be
instantiated by the system, that is needs maker function to set up
database etc.)</li>
</ul>
<p>The rationale behind this was that users should be shielded from other
users' contents and when searching can choose to only gradually
incorporate more search results. However, it makes the whole thing more
complex, database queries potentially less powerful (although Datomic is
quite good with this) and we can separate data in the user interface
while they are not separated in the DB. See also
<a href="considerations">considerations</a>.</p>
