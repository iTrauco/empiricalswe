<p>##Other Changes
There have been some important structural changes to all the ReadyThemes. 9.0005 changed how frameworks get exported and all files included in a framework must now be included in the mm5 folder.</p>
<p>This means that any file that used to be in the web root will get moved to the mm5 folder when the framework is installed in a store.</p>
<p>Because of this change the latest ready themes have been restructured to support this update. We also took this opportunity to reorganize the files included in each theme and create a new "themes" folder.</p>
<p>Each theme has the following changes:</p>
<ol>
<li>
<p>All assets of a theme (CSS, JS, Images, etc) are now in a themes folder: /mm5/themes/theme-name/</p>
<p>Example: /mm5/themes/suivant or /mm5/themes/retro</p>
</li>
<li>
<p>There is now a global variable defined in the head tag of each them which determine the "theme path". This path is where all files are located and is relative to the base href (mm5 folder).</p>
<pre><code>&lt;mvt:assign name="g.theme_path" value="'themes/suivant'" /&gt;
</code></pre>
<p>This variable is then used as a prefix for any assets you want to include in the theme.</p>
<pre><code>&lt;script src="&amp;mvte:global:theme_path;/js/modernizr.js"&gt;&lt;/script&gt;
&lt;script src="&amp;mvte:global:theme_path;/js/respond.min.js"&gt;&lt;/script&gt;
</code></pre>
<p>There is also a JavaScript version of this variable which can be used in JavaScript files:</p>
<pre><code>&lt;script&gt;var theme_path = "&amp;mvtj:global:theme_path;";&lt;/script&gt;
</code></pre>
</li>
</ol>
