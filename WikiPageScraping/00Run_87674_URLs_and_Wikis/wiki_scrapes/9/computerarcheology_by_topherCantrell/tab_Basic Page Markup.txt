<a aria-hidden="true" class="anchor" href="#blocks" id="user-content-blocks"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>Blocks</h1>
<p>A line that begins with <code>{{{</code> marks the beginning of a block of lines. The block ends with a line that contains only <code>}}}</code>.</p>
<p>Types of blocks:</p>
<ul>
<li>
<code>{{{</code> A &lt;pre&gt; formatted area (no &lt;p&gt; elements are added)</li>
<li>
<code>{{{HTML</code> A block of pure HTML that is not processed at all</li>
<li>
<code>{{{PlayMe</code> A block of lines given the "playMe" styling class</li>
<li>
<code>{{{Memory</code> A block of lines that contain tables of memory mappings (more on this below)</li>
</ul>
<p>Other tokens on the end of the braces are used to match CSS styling. These generate regular, plain <code>{{{</code> described above but with class style. For instance:</p>
<ul>
<li>
<code>{{{plainCode</code> matches class style <code>block_plainCode</code>
</li>
</ul>
