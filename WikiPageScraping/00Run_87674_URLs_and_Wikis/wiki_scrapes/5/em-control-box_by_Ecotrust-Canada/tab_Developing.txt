<h2>
<a aria-hidden="true" class="anchor" href="#disk-layout" id="user-content-disk-layout"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>Disk Layout</h2>
<p>The control box has 2 disks, an OS disk, and optional DATA disk. The OS disk contains at least 2 partitions, labelled BOOT and VAR. Boxes with a development environment installed also have a DEV partition. The purpose of these is as follows:</p>
<ul>
<li>BOOT is not mounted in a running system. It's used by the bootloader to launch either the development or production EM operating system. It contains a bootloader config at "/grub/grub.conf" and any number of software images, such as /em-X.Y.Z which contain the entire production EM system.</li>
<li>VAR is a run-time data directory. When a production EM system runs, the root directory (/) is read-only, and is loaded from the image file. Any files that need to be editable in the production system are stored here. The VAR partition is mounted at the /var location.</li>
<li>DEV is the development environment. It may be used in place of a read-only production image.</li>
</ul>
<h2>
<a aria-hidden="true" class="anchor" href="#file-layout" id="user-content-file-layout"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>File Layout</h2>
<p><code>/opt</code> - custom software
<code>/opt/em</code> - the <a href="https://github.com/Ecotrust-Canada/em-control-box">EM software</a>
<code>/opt/elog</code> - the Elog software (optional)
<code>/var</code> - the run-time data directory
<code>/var/em</code> - editable EM files, like config.
<code>/var/em/data</code> - backup data storage. data recorded is cached here before flushing to /mnt/data, in case the data disk is missing and for a backup. Only a short period of video is stored here at a time due to space constraints.
<code>/var/elog</code> -
<code>/mnt/data</code> - the data disk mount point, all data recorded is stored here.
<code>/etc/em.conf</code> - the EM system config file. links to /var/em/em.conf in order to be writable.
<code>/etc/systemd/system</code> - startup scripts, which defined dependencies and processes on boot
<code>/tmp/em-state.json</code> - a JSON api that can be read to get the system state from the filesystem any time. Used by the web interface (UI) as well.</p>
