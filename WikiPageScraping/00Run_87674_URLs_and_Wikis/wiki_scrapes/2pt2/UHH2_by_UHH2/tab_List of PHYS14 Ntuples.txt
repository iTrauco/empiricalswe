<p>All datasets are available as <code>xml</code> file snippets (to be included in the main sframe <code>xml</code> configuration) in git at <code>UHH2/common/datasets</code>. Please refer to the git repository for an up-to-date list of the available samples. Note that this page does not exhaustively list the currently available samples; rather, it provides a “translation” from the xml filenames to the dbs sample name and provides other dataset-related information such as cross sections and instructions of how to generate the xml files.</p>

<a aria-hidden="true" class="anchor" href="#creating-xml-files" id="user-content-creating-xml-files"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>Creating xml files</h1>
<p>After processing a dataset, create the corresponding xml file for SFrame with the script UHH2/scripts/create-dataset-xmlfile. This script takes the filename pattern as first argument and the output xml filename as second. Make sure to use quotation marks to prevent the shell from expanding the pattern. For example:
  
</p><pre lang=""><code>./create-dataset-xmlfile '/nfs/dust/cms/user/nowatsd/grid_control_new/gc-output/PHYS14v1-1/backgrounds/MC_TTJets_MSDecaysCKM_central_20x25_*_Ntuple.root' MC_TTJets_MSDecaysCKM_central_20x25.xml</code></pre>

<p>Add the xml file to git, either in the analysis repository or centrally in UHH2/common/datasets.</p>
