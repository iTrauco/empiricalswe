<p>Drawing an image is best done by using the <a href="https://www.libsdl.org/projects/SDL_image/" rel="nofollow">SDL_image</a> module. It can loads several famous formats.</p>
<h2>
<a aria-hidden="true" class="anchor" href="#loading-sdl_image" id="user-content-loading-sdl_image"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>Loading SDL_image</h2>
<p>To load SDL_image, please be sure that you compiled SDL_image with Lua-SDL2. They are built as separate libraries and needs to be loaded by hand with <a href="http://www.lua.org/manual/5.2/manual.html#pdf-require" rel="nofollow">require</a></p>
<pre><code>local image = require "SDL.image" 
</code></pre>
<h2>
<a aria-hidden="true" class="anchor" href="#initializing-sdl_image" id="user-content-initializing-sdl_image"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>Initializing SDL_image</h2>
<p>To initialize SDL_image, calls <a href="Image-init">image.init</a> and pass one or more image.flags for the desired formats. The functions returns a table with loaded formats and an optional error message.</p>
<pre><code>local formats, ret, err = image.init { image.flags.PNG }

if not formats[image.flags.PNG] then
error(err)
end
</code></pre>
<h2>
<a aria-hidden="true" class="anchor" href="#load-the-image" id="user-content-load-the-image"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>Load the image</h2>
<p>To load an image, you can use <a href="Image-load">image.load</a> and specify the image by path. This functions returns a <a href="Sdl-SurfaceObject">Surface</a>.</p>
<pre><code>local img, ret = image.load("Lua-SDL2.png")
if not img then
error(err)
end
</code></pre>
<h2>
<a aria-hidden="true" class="anchor" href="#create-a-renderer" id="user-content-create-a-renderer"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>Create a renderer</h2>
<p>To create a 2D accelerated context, one should create a <a href="Sdl-RendererObject">Renderer</a>.</p>
<pre><code>local rdr, err = SDL.createRenderer(win, 0, 0)
if not rdr then
error(err)
end
</code></pre>
<h2>
<a aria-hidden="true" class="anchor" href="#convert-to-texture" id="user-content-convert-to-texture"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>Convert to texture</h2>
<p>Before rendering, you need to convert the surface to a <a href="Sdl-TextureObject">Texture</a></p>
<pre><code>img = rdr:createTextureFromSurface(img)
</code></pre>
<h2>
<a aria-hidden="true" class="anchor" href="#draw-to-renderer" id="user-content-draw-to-renderer"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>Draw to renderer</h2>
<p>Now that you converted the surface to a texture you can copy it.</p>
<pre><code>for i = 1, 50 do
    rdr:setDrawColor(0xFFFFFF)
    rdr:clear()
    rdr:copy(img)
    rdr:present()

    SDL.delay(100)
end
</code></pre>
