          <p>#REST API
The most common way to interact with MSViz backend is via the REST API. The full documentation is described by swagger under http://YOUR_HOST/docs/index.html.
Nevertheless, here are the main command, given that your server is in development mode.</p>
<h3>
<a aria-hidden="true" class="anchor" href="#sequences" id="user-content-sequences"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>Sequences</h3>
<p>There are two very close url: /sequences for command outching several entries at once (all of a source, for example) or /sequence for the one targeting more precisely an entry</p>
<p>####Load a fasta file</p>
<pre><code>curl -X POST                                   \
     --data-binary=@test/resources/M_100.fasta  \
     http://localhost:9000/sequences/M_100/fasta
</code></pre>
<p>####List all sources
All the source tags (from the <code>source</code> argument in the load call).</p>
<pre><code>curl http://localhost:9000/sequences/list-sources
</code></pre>
<p>####Get a protein</p>
<pre><code>curl http://localhost:9000/sequence/M_100/Q09666
</code></pre>
<p>####Delete a whole source</p>
<pre><code>curl -X DELETE http://localhost:9000/sequences/M_100
</code></pre>

</code></pre>
