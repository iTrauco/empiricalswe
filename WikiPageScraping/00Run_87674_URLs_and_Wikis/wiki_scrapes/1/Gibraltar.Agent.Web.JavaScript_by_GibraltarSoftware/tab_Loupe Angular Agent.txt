          <p>#Loupe Angular Agent</p>
<p>The Angular agent has been designed for use in any Angular 1.x solution.</p>
<p>It provides logging functionality via a <code>Loupe.LogService</code> and adds a decorator to the Angular <code>$exceptionHandler</code> service for unhandled exceptions.</p>
<h2>
<a aria-hidden="true" class="anchor" href="#unhandled-exceptions" id="user-content-unhandled-exceptions"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>Unhandled exceptions</h2>
<p>As mentioned above the angular agent decorates the normal angular <code>$exceptionHandler</code> service so that if an error is throw it will be called by angular
and take details of the exception and log them to the server.</p>

<p>See <a class="internal present" href="/GibraltarSoftware/Gibraltar.Agent.Web.JavaScript/wiki/Logging">Logging</a> for details on the methods available to log and details on how to log.</p>
<h2>
<a aria-hidden="true" class="anchor" href="#services" id="user-content-services"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>Services</h2>
<p>The agent has 3 services:</p>
<ol>
<li>loupe.logService</li>
<li>loupe.stacktraceService</li>
<li>loupe.platformService</li>
</ol>
<p>Both <code>loupe.stacktraceService</code> and <code>loupe.platformService</code> can be used independently of the <code>loupe.logService</code> if you so wish.</p>
<h4>
<a aria-hidden="true" class="anchor" href="#loupelogservice" id="user-content-loupelogservice"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>loupe.LogService</h4>
<p>This is the service to use for logging messages to the server</p>

<p>Wraps <a href="http://www.stacktracejs.com/" rel="nofollow">stackTrace.js</a> and provides the stackTrace.js functionality</p>
<h4>

<p>Wraps <a href="https://github.com/bestiejs/platform.js">platform.js</a> and provides access to its functionality</p>
<h2>
<a aria-hidden="true" class="anchor" href="#loupe-objects" id="user-content-loupe-objects"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>Loupe Objects</h2>
<p>As well as the services the agent also exposes a couple of loupe specific "objects":</p>
<ol>
<li>logMessageSeverity - an object literial for specifying the serverity on a log messate</li>
<li>MethodSourceInfo - object to record where a message was logged from see <a class="internal present" href="/GibraltarSoftware/Gibraltar.Agent.Web.JavaScript/wiki/MethodSourceInfo">MethodSourceInfo</a>
</li>
</ol>
<p>Both of these are accessed via the logService e.g. <code>logService.write(logService.logMessageSeverity.Verbose,"My Category", "My Caption")</code> and
<code>var info = new logService.MethodSourceInfo("app.js")</code></p>
