          <p><code>indexes.py</code> houses the core classes wrapping Google's search API. There's an <code>Index</code> class that provides the interface for creating indexes and adding, removing and searching documents. There's also the <code>DocumentModel</code> class, subclasses of which define the documents that can be added to and searched through indexes.</p>

</h4>
<p>Lists the documents (or only their doc ids if <code>ids_only=True</code>) indexed with this index in <code>doc_id</code> order. If <code>start_doc_id</code> is given, the index will start listing documents from that id (much like a cursor) and the start doc can be skipped by setting <code>include_start_doc=False</code>. The maximum number of documents returned is 1000.</p>

<h4>
<a aria-hidden="true" class="anchor" href="#searchdocument_class-ids_onlyfalse" id="user-content-searchdocument_class-ids_onlyfalse"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a><code>search(document_class, ids_only=False)</code>
</h4>
<p>Returns a <code>SearchQuery</code> that will search the index and return objects of <code>document_class</code> instantiated from the search results. If <code>ids_only</code> is true, a list of document ids matching the query will be returned instead.</p>
<p>It is required to pass the <code>document_class</code> explicitly, because the index isn't aware of what type the documents are. There's nothing to stop multiple document types being indexed with the same index so it doesn't assume here what the type you want back is.</p>
<p>This might seem initially weird - why only allow instantiation of one class if you can index more than one document type? Because filtering and ordering multiple document types by the same criteria rarely makes sense. See <a href="link_pending">global search indexes</a> for a suggestion on how to create an index that spans multiple content types.</p>
