<h2>
<a aria-hidden="true" class="anchor" href="#1-code-base" id="user-content-1-code-base"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>1. Code base</h2>
<p>Create a new project using <a href="https://github.com/grrr-amsterdam/garp_scaffold">Garp Scaffold</a>.<br/>
Or <code>git clone</code> an existing one.<br/>
Run <code>composer install</code> to grab all dependencies.</p>
<h2>
<a aria-hidden="true" class="anchor" href="#2-database" id="user-content-2-database"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>2. Database</h2>
<p>Create a UTF-8 MySQL database and call it <em>project name</em> + "_d". The name has to match the setting in <code>/application/configs/db.ini</code>:</p>
<pre><code>[development]
...
resources.db.params.dbname = my_project_name_d
</code></pre>
<p>In your MySQL client, allow the Garp database user access to the database.</p>
<ul>
<li>User: <code>garp_development</code>
</li>
<li>Pass: <code>welovegarp</code>
</li>
</ul>

Assets</h2>
<p>Prepare the CDN. Make sure your asset files are distributed to S3:</p>
<pre><code>$ g cdn distribute --to=development
</code></pre>

Controller</h2>
<p>Create some controller actions. If you're creating a new controller, make sure to add it to <code>/application/configs/acl.ini</code> as well. See <a href="http://code.grrr.nl/garp3/wiki/acl" rel="nofollow">ACL</a> for more on that.</p>

Routes</h2>
<p>Make sure the url you're trying out is mentioned in <code>/application/configs/routes.ini</code></p>

Views</h2>
<p>You'll need a view to render a template. Within <code>/application/modules/default/views/scripts/</code> there should be a folder per controller. Inside should be a <em>.phtml</em> file, bearing the name of the controller action.</p>

Snippets</h2>
<p>Create some snippets in <code>/application/configs/snippets.ini</code> and call the snippet generator:</p>
<pre><code>$ g snippet create from file
</code></pre>
