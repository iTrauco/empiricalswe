<a aria-hidden="true" class="anchor" href="#raw-lens" id="user-content-raw-lens"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>Raw Lens?</h2>
<p>What I mean by "raw lens" is a Lens constructed via its constructor instead of a <code>nanoscope</code> method. The main benefit of doing this is to <strong>create Lenses that focus on a sub-part of an object without access to that object directly</strong>.</p>
<p>For example, we could peer into the second element of an array in this manner, using <code>nanoscope.index</code>:</p>
<div class="highlight highlight-source-js"><pre><span class="pl-k">var</span> arr <span class="pl-k">=</span> [<span class="pl-c1">1</span>, <span class="pl-c1">2</span>, <span class="pl-c1">3</span>, <span class="pl-c1">4</span>];
<span class="pl-k">var</span> lens <span class="pl-k">=</span> <span class="pl-en">nanoscope</span>(arr).<span class="pl-c1">index</span>(<span class="pl-c1">1</span>);

<span class="pl-smi">lens</span>.<span class="pl-c1">get</span>();
<span class="pl-c"><span class="pl-c">//</span> #=&gt; 2</span></pre></div>
<p>However, what if we don't have access to <code>arr</code>? What if we want a generic Lens into the second element <em>of any array imaginable</em>? This is where the <code>Lens</code> constructors come in:</p>
<div class="highlight highlight-source-js"><pre><span class="pl-k">var</span> lens <span class="pl-k">=</span> <span class="pl-k">new</span> <span class="pl-en">nanoscope.IndexedLens</span>(<span class="pl-c1">1</span>);

<span class="pl-smi">lens</span>.<span class="pl-en">view</span>([<span class="pl-c1">1</span>, <span class="pl-c1">2</span>, <span class="pl-c1">3</span>, <span class="pl-c1">4</span>]).<span class="pl-c1">get</span>();
<span class="pl-c"><span class="pl-c">//</span> #=&gt; 2</span></pre></div>
<p>This is a bit more verbose, but very powerful -- with this functionality, it becomes possible to create reusable Lenses that don't care about their input. In the same sense that <code>nanoscope</code> allows us to perform a wide array of operations on any single input, raw Lenses allow us to perform the same operation on a wide array of inputs.</p>
<p>Note: The chainable utility functions of <code>nanoscope</code> all use these "raw Lenses" under the hood. Below is a reference list of which functions map to which Lenses internally.</p>
