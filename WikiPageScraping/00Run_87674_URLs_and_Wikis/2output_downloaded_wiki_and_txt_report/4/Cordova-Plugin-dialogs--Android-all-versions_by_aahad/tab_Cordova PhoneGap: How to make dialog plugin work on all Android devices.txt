<a aria-hidden="true" class="anchor" href="#cordovaphonegap-how-to-make-dialog-plugin-work-on-all-android-devices" id="user-content-cordovaphonegap-how-to-make-dialog-plugin-work-on-all-android-devices"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>Cordova/PhoneGap: How to make dialog plugin work on all Android devices</h1>
<hr/>
<p><strong>Since Oct 16, 2014</strong> (commit: 23bebf96f7a3388c7483afc0091faecf943692d3), the <strong>dialog Plugin supports API 17 on-wards.</strong></p>
<p>This commit was to <strong>properly</strong> format <strong>right-to-left</strong> and <strong>left-to-right languages</strong>, as shown below in the snapshot. (<a href="https://github.com/apache/cordova-plugin-dialogs/commit/23bebf96f7a3388c7483afc0091faecf943692d3">Check for setTextDirection API level</a>)</p>
<p>Example: without any code modification</p>
<p><img alt="Dialog Plugin running on API 17" src="https://github.com/aahad/Cordova-Plugin-dialogs--Android-all-versions/raw/master/snapshots/dialog_API_17.jpeg"/></p>
<p>Now if there is <strong>NO</strong> need to properly format localized strings (or <strong>no use of localization at all</strong>) then if these lines are <strong>removed</strong> then the plugin support will be available from <strong>API 14 on-wards</strong>.</p>
<h2>
<a aria-hidden="true" class="anchor" href="#bring-down-to-api-14-or-11" id="user-content-bring-down-to-api-14-or-11"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>Bring down to API 14 or 11</h2>
<p>Currently dialog plugin is using device's light theme (<code>AlertDialog.THEME_DEVICE_DEFAULT_LIGHT</code>) that is available since <strong>API 14</strong>. <a href="http://developer.android.com/reference/android/app/AlertDialog.html#THEME_DEVICE_DEFAULT_LIGHT" rel="nofollow">http://developer.android.com/reference/android/app/AlertDialog.html#THEME_DEVICE_DEFAULT_LIGHT</a></p>
<p>So if it can be <strong>changed</strong> with something else like <code>AlertDialog.THEME_HOLO_LIGHT</code>, <code>AlertDialog.THEME_HOLO_DARK</code>  or <code>AlertDialog.THEME_TRADITIONAL</code> as <strong>these</strong> are present <strong>since API 11</strong>. In this way we can make the <strong>plugin work on API 11</strong>. I tried <code>AlertDialog.THEME_HOLO_LIGHT</code> and found exactly <strong>same results</strong> as of with <code>AlertDialog.THEME_DEVICE_DEFAULT_LIGHT</code>.</p>

<p>Using <code>AlertDialog.THEME_HOLO_DARK</code>, I see following result</p>
<p><img alt="Dialog with holo dark theme" src="https://github.com/aahad/Cordova-Plugin-dialogs--Android-all-versions/raw/master/snapshots/dialog_API14_holodark.jpeg"/></p>
<h2>
<a aria-hidden="true" class="anchor" href="#bring-down-to-api-7" id="user-content-bring-down-to-api-7"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>Bring down to API 7</h2>
<p><code>AlertDialog.Builder(Context context, int theme)</code> was used to create instance of AlertDialog and this <strong>two parameter based constructor</strong> was added in <strong>API 11</strong>. While the basic original AlertDialog <strong>constructor <strong>that takes <strong>single parameter</strong> was present since</strong> API 1</strong>. So if there is <strong>NO hard</strong> need for a <strong>specific theme</strong> then by skipping the two parameter based constructor and by using single parameter based constructor we can easily get down to <strong>API 7</strong>.</p>
