          <p>The <code>owtf_testing</code> framework does not remove the output generated by <code>OWTF</code> itself. That decision has been made in order to make it easier for the user to spot any possible problem. It is still possible to remove the output using a bash redirection though.</p>

<p>The output is composed of different blocks where each of them represent the state of a test case:</p>
<ul>
<li>Name of the test method as well as its file path.</li>
<li>An explicit sentence describing in a couple of words what is the purpose of the test.</li>
<li>The status of the test:
<ul>
<li>
<code>OK</code> if the test succeeded</li>
<li>
<code>FAIL</code> if the test has fail (due to unsatisfied assertions)</li>
<li>
<code>ERROR</code> if the test has encountered runtime error (usually followed by a python traceback)</li>
</ul>
</li>
</ul>
