<a aria-hidden="true" class="anchor" href="#logging" id="user-content-logging"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>Logging</h1>
<p>As per the .Net agents the agent has the same methods to allow you to send a log message to the server:</p>
<ul>
<li>verbose</li>
<li>information</li>
<li>warning</li>
<li>error</li>
<li>critical</li>
<li>write</li>
</ul>
<p>With the exception of the <code>write</code> method the methods support the following arguments:</p>
<ol>
<li>category</li>
<li>caption</li>
<li>description</li>
<li>parameters</li>
<li>exception</li>
<li>details</li>
<li>methodSourceInfo</li>
</ol>
<p>The write method differs in that it has additional parameter, severity, which is its first parameter and needs to be supplied by yourself from the object literal
<code>loupe.logMessageSeverity</code>, or <code>logService.logMessageSeverity</code> in the angular agent, the other methods will supply the severity themselves</p>

<h2>
<a aria-hidden="true" class="anchor" href="#logging-a-message" id="user-content-logging-a-message"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>Logging a message</h2>
<p>Since JavaScript does not support method overloading there is only one method definition but you can provide as many, or as few,
paramters as you wish.</p>
<p>In general any parameters you do not supply will be set to null and details will not be logged on the server when it receives the message.</p>
<p>To log a message you call the appropriate method providing the information you wish to log e.g. (examples from <a class="internal present" href="/GibraltarSoftware/Gibraltar.Agent.Web.JavaScript/wiki/Loupe-JavaScript-Agent">Native Agent</a>)</p>
<ul>
<li>Logging startup up of client - <code>loupe.information("Client","Startup")</code>
</li>
<li>Logging details of validation failure - <code>loupe.warning("Client","Invalid Address","User entered invalid address: {0}","I live here")</code>
</li>
<li>Logging custom exception -</li>
</ul>
<pre><code>
    var customError = new Error("caught exception");

    loupe.error("MyClient","Caught exception","Caught exception performing logic", null, customError);
</code></pre>
