          <p>There are three tiers to the MailSlurper application: <strong>Web Server</strong>, <strong>Service Tier</strong>, and the <strong>SMTP Server</strong>. These tiers separate concerns as follows.</p>
<h3>
<a aria-hidden="true" class="anchor" href="#web-server" id="user-content-web-server"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>Web Server</h3>
<p>This is the server for the user interface. All HTML pages and static assets are served from this tier. The web application is broken into three primary pages consisting of the email interface, the saved searches manager, and the settings page. Each of these pages makes heavy use of JavaScript when interacting with the interface. Subsequently JavaScript is used to perform activities such as retrieving email and paging by making AJAX calls to the <strong>Service Tier</strong>.</p>
<h3>
<a aria-hidden="true" class="anchor" href="#service-tier" id="user-content-service-tier"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>Service Tier</h3>
<p>This tier is another HTTP server which offers RESTful style endpoints that facilitate tasks such as getting email collections, details, settings, and more. This tier is considered the <em>API</em>.</p>
<h3>
<a aria-hidden="true" class="anchor" href="#smtp-server" id="user-content-smtp-server"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>SMTP Server</h3>
<p>This is the meat and potatoes of MailSlurper. The SMTP Server is responsible for listening on a port for incoming SMTP client connections, servicing those connections, and storing captured emails into your storage engine.</p>
