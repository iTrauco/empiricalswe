<h2>
<a aria-hidden="true" class="anchor" href="#about" id="user-content-about"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>About</h2>
<p>This is a native (no X11) OS X 10.6 version of <a href="http://gpodder.org" rel="nofollow">gPodder</a> as a standalone application.
It builds on the good work from jralls on <a href="http://live.gnome.org/GTK+/OSX" rel="nofollow">GTK+/OSX</a>, to build a native GTK stack on OS X and package a GTK application as an Mac OS X application.
And of course, it wouldn't be there for the existence of gPodder itself, thanks to Thomas Perl and the other gPodder contributors.</p>

<ul>
<li>a Mac with a 64 bits x86 hardware (eg. Intel Core 2 Duo);</li>
<li>Snow Leopard (10.6) or later.</li>
</ul>
<h2>
<a aria-hidden="true" class="anchor" href="#instructions" id="user-content-instructions"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>Instructions</h2>
<ol>
<li>Download the zip from <a href="https://sourceforge.net/projects/gpodder/files/macosx/" rel="nofollow">here</a>.</li>
<li>Unzip and see the friendly violet icon</li>
<li>Double-click to run, and if it works, you might want to copy it to <em>Applications</em>
</li>
</ol>
<p><strong>If you had an older version of gPodder (2.xx)</strong>, you'll have to upgrade your configuration.</p>
<p>To do so:</p>
<ol>
<li>
<p>Remove the <code>gPodder</code> directory that just appeared in your home directory;</p>
</li>
<li>
<p>open a Terminal (Application &gt; Utilities &gt; Terminal);</p>
</li>
<li>
<p>drag and drop the gPodder application in the Terminal window;</p>
</li>
<li>
<p>remove the whitespace at the end of the command line by hitting backspace and add <code>/Contents/MacOS/gpodder-migrate2tres</code>. To be sure, remove the end of <code>migrate2tres</code> and hit tab to see it completed</p>
</li>
<li>
<p>hit enter</p>
</li>
<li>
<p>You should be presented with such text :</p>
</li>
</ol>
<blockquote>
<p>Would carry out the following actions:</p>
<pre><code> Move downloads from /Users/whatsyourname/gpodder-downloads
                  to /Users/whatsyourname/gPodder/Downloads

 Convert database from /Users/whatsyourname/.config/gpodder/database.sqlite
                    to /Users/whatsyourname/gPodder/Database
</code></pre>
<p>Continue? (Y/n)</p>
</blockquote>
<ol>
<li>Then type <code>y</code> and press Enter. The script will end by saying</li>
</ol>
<blockquote>
<p>Done. Have fun with gPodder 3!</p>
</blockquote>
