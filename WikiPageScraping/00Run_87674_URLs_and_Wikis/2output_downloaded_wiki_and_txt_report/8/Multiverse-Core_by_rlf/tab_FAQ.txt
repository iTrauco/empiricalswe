<p>Multiverse-Core is a World management plugin. What this means is you can use it to create, delete and manage your different worlds in a Bukkit Server. See the <a class="internal present" href="/rlf/Multiverse-Core/wiki/basics">Basic Multiverse Tutorial</a> for a very brief overview!</p>
<h2>
<a aria-hidden="true" class="anchor" href="#why-does-it-have-the--core-at-the-end" id="user-content-why-does-it-have-the--core-at-the-end"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a><a href="#wiki-core" name="user-content-core">•</a>Why does it have the '-Core' at the end?</h2>
<p>MultiVerse (note the capital V which is no longer in the name Multiverse-Core) was the old version, but a lot has changed. Multiverse-Core is a <em>part</em> of the <a href="https://github.com/Multiverse/Multiverse-Core/wiki/Plugins-of-Multiverse-2">Multiverse 2 package</a>. What we decided to do was break up MultiVerse into <a href="https://github.com/Multiverse/Multiverse-Core/wiki/Plugins-of-Multiverse-2">several smaller plugins</a>. This way we avoid plugin conflicts. For example, if you, the server admin, want to use a different portal plugin because you think ours sucks, you now can install just <a href="https://github.com/Multiverse/Multiverse-Core/wiki">Multiverse-Core</a>.</p>

<p>Mojang removed the skyland code in Minecraft 1.0.0, replacing it with "the End" chapter of the game.  That included (1) floating island terrain, (2) a gray noontime sky, (3) clouds drawn at very low Y levels.  You have two choices.</p>
<p><strong>Choice A:</strong>  Use a terrain generator plugin like <a href="http://dev.bukkit.org/server-mods/skylandsplus/" rel="nofollow">SkylandsPlus</a> to allow players to continue to explore by generating additional chunks. This may cause tearing at the explored edges as it's not the same generator.</p>
<p><strong>Choice B:</strong>  If your skylands were already pretty mapped out, then use a border plugin like <a href="http://dev.bukkit.org/server-mods/worldborder/" rel="nofollow">WorldBorder</a> to stop any more expansion of the world.</p>
<p>In either case, edit your world configuration to use a <code>NORMAL</code> environment instead of the old missing <code>SKYLAND</code> environment type.  Refine your monster spawning settings to suit your tastes.  If you want permanent day, plugins like <a href="http://dev.bukkit.org/server-mods/timeshift/" rel="nofollow">TimeShift</a> can help you with that, too.</p>

<p>Multiverse has added a new property, <code>mode</code>, to the list of properties on a world. This means you can set the mode on a per-world basis! To change a world to creative, type:</p>
<pre><code>/mvm set mode creative
</code></pre>
<p>The world will become a creative world, and any players will be set to creative mode when they enter the world. To set it back, simply type:</p>
<pre><code>/mvm set mode survival
</code></pre>
<p>Keep in mind that <strong>ops can enter creative mode in a survival world</strong> with the <code>/gamemode</code> command. Be careful to only op players that you trust!</p>

<p>The teleport command was becoming immensely powerful and some people don't want their users to have all this power. It is a little bit more complex, but allows for much more flexible teleportation.</p>
<p>Now, Each <a class="internal present" href="/rlf/Multiverse-Core/wiki/Destinations">Destination</a> type gets it's own 2 permissions (These are listed on the <a class="internal present" href="/rlf/Multiverse-Core/wiki/Destinations">Destinations</a> page). You can still give a user <code>multiverse.teleport.*</code> to give them global teleport access for themself <strong>and</strong> others.</p>
<p>If you just want them to have teleport rights to every destination type for themself: <code>multiverse.teleport.self.*</code><br/>
<em><strong>HINT:</strong> This is the same as the old <code>multiverse.core.tp.self</code> permission!</em></p>
<p>This also allows for very advanced setups like "I want people to be able to teleport <strong>only themselves</strong> everywhere <strong>except</strong> other players! Cuz that's cheating!":</p>
<p>With PermissionsBukkit (just an example, you should be able to do the same with your perms plugin)</p>
<pre><code>&gt; perm group setperm moderator multiverse.teleport.other.* false
09:56:46 [INFO] Group moderator now has multiverse.teleport.other.* = false.

&gt; perm group setperm moderator multiverse.teleport.self.* true
09:56:47 [INFO] Group moderator now has multiverse.teleport.self.* = true.

&gt; perm group setperm moderator multiverse.teleport.self.pl false
09:56:48 [INFO] Group moderator now has multiverse.teleport.self.pl = false.
</code></pre>

<a aria-hidden="true" class="anchor" href="#multiverse-crashes-my-server-while-adjusting-my-spawn-stop-it" id="user-content-multiverse-crashes-my-server-while-adjusting-my-spawn-stop-it"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a><a href="#wiki-adjustspawn" name="user-content-adjustspawn">•</a>Multiverse crashes my server while adjusting my spawn! Stop it!</h2>
<p>The latest development builds include extra options to stop Multiverse from attempting to auto-adjust the spawn point, which is especially useful in Skylands or empty (NullTerrain) worlds. Generating a new world without spawn adjustment is a two-step process.</p>
<p>First, create the world with the normal <code>mv create</code> command, but add <code>-n</code> to the end of it. For NullTerrain, this might look something like:</p>
<pre><code>mv create empty_world normal -g NullTerrain -n
</code></pre>
<p>The <code>-g</code> flag indicates that Multiverse should use the NullTerrain generator, and the <code>-n</code> flag tells Multiverse to <em>not</em> adjust the spawn for this world. Second, once the world is created, immediately run the following command to stop Multiverse from adjusting the spawn in the future:</p>
<pre><code>mvm set adjustspawn false empty_world
</code></pre>
<p>This will tell Multiverse to <em>never</em> adjust the spawn, even on world reload or server restart.</p>

<p>New builds of Multiverse-Core include an extra permission to prevent automatic mode-switching. This permission takes the form:</p>
<pre><code>mv.bypass.gamemode.WORLD_NAME
</code></pre>
<p>With this new permission, server owners can pick particular users to ignore some (or all) of the game mode settings on each world when teleporting between them. For example, a player with <code>mv.bypass.gamemode.*</code> could be an admin, keeping their creative mode setting even in survival worlds. Just set the right permissions in your preferred plugin and you're good to go.</p>
