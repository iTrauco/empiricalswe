<h2>
<a aria-hidden="true" class="anchor" href="#introduction" id="user-content-introduction"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a><a href="#wiki-intro" name="user-content-intro">•</a>Introduction</h2>
<p>In Multiverse 2, we have included a handy new help system. To access this help, simply type:</p>
<p><a href="#help"><code>/mv</code></a></p>
<p>In order to properly read this command reference, we'll walk through reading the "usage" of a command.</p>
<h4>
<a aria-hidden="true" class="anchor" href="#command-case" id="user-content-command-case"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>Command Case</h4>
<p>Whenever reading this wiki or the in game <code>[/mv](#help)</code> help, you should remember that items in <strong>ALL CAPPS</strong> should be replaced with a variable. Items in lowercase should be typed exactly how they appear.</p>
<h4>
<a aria-hidden="true" class="anchor" href="#optional-vs-required-params" id="user-content-optional-vs-required-params"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>Optional vs. Required Params</h4>
<p>Items inside square braces (<code>[]</code>) are <strong>OPTIONAL</strong> where items inside curly braces (<code>{}</code>) are <strong>REQUIRED</strong></p>
<h4>
<a aria-hidden="true" class="anchor" href="#simple-command-example" id="user-content-simple-command-example"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>Simple command example</h4>
<p>Let's look at a simple one first:</p>
<p><code>[/mv info [WORLD]](#wiki-info)</code></p>
<p>This means that this command has one optional parameter, meaning you could type <a href="#wiki-info"><code>/mv info</code></a> <strong>or</strong> <a href="#wiki-info"><code>/mv info world</code></a>. Obviously the command will do different things with or without it's params. You can tell what those effects are by reading this wiki.</p>
<h4>
<a aria-hidden="true" class="anchor" href="#complex-command-example" id="user-content-complex-command-example"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>Complex command example</h4>
<p>Let's look at the most complex example:<br/>
<em>NOTE: This is the most complex command in MV2 and in order to use it correctly, please refer to the <a href="#wiki-create">Create Command</a>.</em></p>
<p><a href="#wiki-create"><code>/mv create {NAME} {ENV} -s [SEED] -g [GENERATOR[:ID]]</code></a></p>
<p>At first you will notice that there are 2 required params, <strong>NAME</strong> and <strong>ENV</strong>. Then you will see the flags afterwards. If you want to use a <a class="internal present" href="/rlf/Multiverse-Core/wiki/World-properties#wiki-seed">seed</a> you must add <code>-s gargamel</code> to the command string. If you want a <a class="internal present" href="/rlf/Multiverse-Core/wiki/World-properties#wiki-generator">generator</a> the same principle applies: <code>-g BukkitFullOfMoon</code>. Full examples of this command are located at the <a href="#wiki-create">Create Command reference</a></p>

<a aria-hidden="true" class="anchor" href="#details" id="user-content-details"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>Details:</h4>
<p>The list command will only show you the worlds that you have access to enter. For example, if you are blacklisted in <code>sky</code> but not in <code>world</code> and <code>world_nether</code>, when you type <code>/mvlist</code> you would see:</p>
<pre><code>Worlds which you can view:
world - NORMAL
world_nether - NETHER
</code></pre>
<p>This allows you to have worlds that are completely unknown to people. Keep in mind though, if you're in a world that someone else cannot view, and you chat from it, your chat prefix would be of that world.

<p>The help command now supports searching! This means if you type <code>/mvh del</code> you will see a list of all commands whose <code>name</code>, <code>description</code>, <code>alias(s)</code> or <code>usage</code> contain 'del'. If you wish to <strong>search</strong> for any commands that contain the word 'delete', use any of the other help aliases: <code>/mvh delete</code>, <code>/mvhelp delete</code>, <code>/mv help delete</code>...</p>

<p>As of minecraft 1.1, the concept of world types has been introduced. These seem to be like generators, but baked into minecraft itself. Currently, there are only <code>FLAT</code>, <code>LARGEBIOMES</code> and <code>NORMAL</code> for world types. To use a world type, simply use the <code>-t</code> parameter, for type. <a aria-hidden="true" class="anchor" href="#map-features-generate-structures" id="user-content-map-features-generate-structures"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a><a href="#wiki-mapfeatures" name="user-content-mapfeatures">•</a>Map Features (Generate Structures)</h4>
<p>As of minecraft 1.1 if you want a flat world with no structures, all you have to do is add the <code>-a false</code> flag. We used <code>-a</code> for <strong>Allow Structures</strong> and because <code>-g</code> was already taken.</p>
<p>*<strong>NOTE:</strong> This will <strong>only</strong> work with <code>NORMAL</code> worlds. @FernFerret has some <a href="https://github.com/Bukkit/CraftBukkit-Bleeding/commit/68897b284e42f8fc0bba85e736a36b5cbbf89fe3#commitcomment-912997">neat code here to allow flat Nether worlds with structures</a> but it has not been added to CraftBukkit. You'll have to build it yourself.</p>
<p><code>/mv create flat_world NORMAL -t FLAT</code></p>
<p>Currently, there are only <code>FLAT</code> and <code>NORMAL</code> for world types.</p>

<h4>
<a aria-hidden="true" class="anchor" href="#generators" id="user-content-generators"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a><a href="#wiki-generators" name="user-content-generators">•</a>Generators</h4>
<p>Multiverse 2 supports Custom Generators. What does this mean? We support any plugin that makes use of the Bukkit custom generator feature. If an author has implemented their own style that does NOT use the custom generator method <a href="http://forums.bukkit.org/threads/22795/" rel="nofollow">described here</a>, it may not work with MV. We will try and keep an updated page of <a class="internal present" href="/rlf/Multiverse-Core/wiki/Custom-Generator-Plugins">Custom Generator Plugins</a>.</p>
<p>Once you've found a plugin you want to use, copy the jar of that plugin to your plugins folder. Then all that's left to do is type a single command:<br/>
(For this example I've decided to use <a href="https://github.com/Dinnerbone/BukkitFullOfMoon">BukkitFullOfMoon</a> which will generate a cratered moon world by @Dinnerbone).</p>
<p><code>/mv create moon NORMAL -g BukkitFullOfMoon</code></p>
<p>The <code>BukkitFullOfMoon</code> above is the plugin name. This plugin only has one generator, and it's defined correctly, so the <code>:GENID</code> is needed. Some plugins may contain multiple generators, in which case you would type this: (This next one I'm making up entirely...)</p>
<p><code>/mv create land_lake NORMAL -g WaterWorlds:Lakes</code></p>
<p>OR</p>
<p><code>/mv create hell_lake NORMAL -g WaterWorlds:LavaLakes</code></p>
<hr/>
<p>####<a name="user-content-spacesnote">***</a>A note about spaces in world names:
You should know that if you decide to use spaces in your world name that whenever people want to use <a href="#wiki-teleport"><code>/mv tp</code></a>, <a href="#wiki-modify"><code>/mv modify</code></a> or any command that takes the world name as a param, they will have to surround the name in quotes, like so:</p>
<p><code>/mv tp "My world name with spaces"</code></p>
<p><code>/mv tp My world name with spaces</code> will NOT work.</p>

<h4>
<a aria-hidden="true" class="anchor" href="#details-8" id="user-content-details-8"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>Details:</h4>
<p>The teleport command allows you to teleport yourself or others to a specified world. There are many different permissions associated with this command, with the categories being <code>self</code> and <code>other</code>; allowing you to teleport yourself and others respectively.</p>
<p>Each Type of <a class="internal present" href="/rlf/Multiverse-Core/wiki/Destinations">Destination</a> will have each of the permissions (<code>self</code> and <code>other</code>) assoicated with it. The most basic example is for worlds:</p>
<pre><code>multiverse.teleport.self.w
multiverse.teleport.other.w
</code></pre>
<p>The letter(s) at the end are the identifier for the destination. This is the same thing you would put before the destination if you teleport to it: <code>/mv tp w:MyWorld</code>. So Portals would be:</p>
<pre><code>multiverse.teleport.self.p
multiverse.teleport.other.p
</code></pre>
<p>Even if you have the <code>multiverse.teleport.other.w</code> permission, you can only teleport people to worlds where <strong>YOU</strong> can go yourself. This will allow you to teleport a player to a world that they themselves cannot go to, as long as you have the permission to do so.</p>
<p>Please see the <a class="internal present" href="/rlf/Multiverse-Core/wiki/FAQ#wiki-tp-perms">FAQ</a> for why this change was made (Old permissions were simply <code>multiverse.core.tp.[self|other]</code>!</p>

<p>Displays who is currently in which worlds. If you want to see who is in a specific world, pass the world name in as the only parameter. By default, <code>/mv who</code> will <strong>not</strong> show worlds that are empty. If you want to see <strong>all</strong> worlds simply add either <code>-a</code> or <code>--all</code> to the end of your command: <code>/mvwho -a</code>. The <code>-a</code>/<code>--all</code> flag will <strong>override</strong> a world provided.</p>

<h4>
<a aria-hidden="true" class="anchor" href="#details-15" id="user-content-details-15"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>Details:</h4>
<p>The Purge command allows you to remove all of a specified type of entity from a world. Understand that if the spawn rules allow the purged type of mob to spawn, they'll populate almost instantly after the purge. This is used to clear out any remaining mobs from a setting change or if a user abuses a mob spawn command (Like the <code>/adopt</code> command from the awesome WolfPound plugin).</p>
<p>If you specify the first parameter as <code>all</code> All worlds will be purged of the specified mob type. You are allowed to specify multiple mob types. Feel free to mix monsters and animals.</p>

<p>The modify command lets you set the variables for your Multiverse worlds in game without needing to ever edit a config file. The <code>set</code> command sets a non-array value. An example would be turning animal spawning on, or setting the world scale to 2. The PROPERTY value that is required for this command must be one of the values listed below. If you do not specify a world, the current world will be used. A world is <strong>required</strong> from the console.</p>
<h5>
<a aria-hidden="true" class="anchor" href="#variables-you-can-set" id="user-content-variables-you-can-set"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>Variables you can set:</h5>
<ul>
<li>alias - String</li>
<li>color - String
<ul>
<li>Must be one of these: <code>AQUA, BLACK, BLUE, DARKAQUA, DARKBLUE, DARKGRAY, DARKGREEN, DARKPURPLE, DARKRED, GOLD, GRAY, GREEN, LIGHTPURPLE, RED, YELLOW, WHITE</code>
</li>
</ul>
</li>
<li>animals - Boolean</li>
<li>monsters - Boolean</li>
<li>pvp - Boolean</li>
<li>scale/scaling - Double</li>
<li>price - How much it costs to enter the world. Enter 0 for free. Enter negative value for debit, positive value for credit.</li>
<li>currency/curr - What it costs to enter the world in the form of an Item ID (-1 to use the econ plugin you have installed)</li>
<li>respawn - String
<ul>
<li>This value can either be the Name of a world or the Alias of that world. Space CAN be used.</li>
</ul>
</li>
<li>mode/gamemode - String
<ul>
<li>Sets the mode for all players in a world. Must be <code>creative</code> or <code>survival</code>
</li>
</ul>
</li>
<li>diff/difficulty - String/Integer
<ul>
<li>Sets the difficulty of a world. You can use string or integers:</li>
<li>Possible values are PEACEFUL - 0, EASY - 1, NORMAL - 2, HARD - 3</li>
</ul>
</li>
<li>weather - Boolean
<ul>
<li>Turn weather on/off.</li>
</ul>
</li>
</ul>
<p>See the <a class="internal present" href="/rlf/Multiverse-Core/wiki/World-properties">World Properties</a> page for more detail on these variables.</p>
<p>The second item is the type required. This means you cannot do <code>/mv modify set scale fish</code> or <code>/mv modify set animals 7</code></p>
<p>#####A note about animals and monsters:
The <code>animals</code> and <code>monsters</code> values when used with <code>set</code> will set the overall spawn behavior of animals/monsters. If you have any monsters in the <code>monsters</code> list and you <code>/mv modify set monsters true</code> you're saying:   "I want monsters to spawn, <strong>except</strong> the ones in the monsters list"</p>
