          <p><strong>Getting Involved and Contributing</strong>
You do not have to be a security expert or a programmer to contribute. The following are some ideas for contributing but if you have other ideas, please send them to a Project Leader.</p>
<p><strong>Coding</strong>
We could implement some of the later items on the roadmap sooner if someone wanted to help out with unit or automated regression tests</p>
<p><strong>Localization</strong>
Are you fluent in another language? Can you help translate the text strings in the Code Project Template into that language?</p>
<p><strong>Testing</strong>
Do you have a flair for finding bugs in software? We want to product a high quality product, so any help with Quality Assurance would be greatly appreciated. Let us know if you can offer your help.</p>
<p><strong>Build Management</strong>
Are you a GitHub ninja? Into build management, setting up GitHub badges, Travis CI, etc. Let us know if you can offer your help.</p>
<p><strong>Feedback</strong>
We are open to any thoughts you have to improve DeepViolet. When you propose your idea be sure to include how you can help. Ideas are great but execution is king. Please send your comments or ideas to the project leader(s) or post it as a project issue.</p>
