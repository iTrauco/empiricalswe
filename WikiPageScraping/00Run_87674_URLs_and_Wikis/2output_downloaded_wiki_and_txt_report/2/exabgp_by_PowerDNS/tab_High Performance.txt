<p>ExaBGP has an undocumented, but useful feature allowing to specify multiple configurations files.</p>
<p>For example :</p>
<pre><code>./bin/exabgp -d exabgp-1.conf exabgp-2.conf
</code></pre>
<p>In that scenario, ExaBGP will re-fork and start one instance per configuration file. It helps reduce a little the memory utilisation when using many configuration files.</p>
