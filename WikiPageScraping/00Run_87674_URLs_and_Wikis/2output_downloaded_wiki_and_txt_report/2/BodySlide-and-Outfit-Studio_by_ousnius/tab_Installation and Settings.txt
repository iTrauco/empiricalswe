          <h2>
<a aria-hidden="true" class="anchor" href="#installation" id="user-content-installation"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>Installation</h2>
<p>Simply download the program from one of its mod pages and extract it to your game's data folder.</p>
<p>Making use of mod managers is recommended, as well as keeping the program in the default folder provided with the download for the specific game, as this will make installing addons easier.</p>
<p>Before trying to configure the program, make sure you have set up a proper installation.</p>
<ul>
<li>When using mod managers, please follow the instructions provided on the mod pages for running BodySlide.</li>
<li>Generally, make sure to read the description of the mod page you downloaded BodySlide from.</li>
</ul>
<h2>
<a aria-hidden="true" class="anchor" href="#target-game" id="user-content-target-game"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>Target Game</h2>
<p>When running the executable for the first time, the following dialog window will show.</p>
<p><a href="http://i.imgur.com/i1K4gn0.png" rel="nofollow"><img alt="Target Game" data-canonical-src="http://i.imgur.com/i1K4gn0m.png" src="https://camo.githubusercontent.com/5abb19d7058666a9d9984bcd2ba8d2085de3f7c3/687474703a2f2f692e696d6775722e636f6d2f69314b34676e306d2e706e67"/></a></p>
<p>Select the "Data" folder of the game you want to use both BodySlide and Outfit Studio for and continue.</p>
<p>Games that are found in the registry will automatically have their paths filled out, but you can only choose one target game at the same time. You can change this selection later on in the settings dialog.</p>

<h3>
<a aria-hidden="true" class="anchor" href="#game" id="user-content-game"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>Game</h3>
<p><strong>Target Game</strong><br/>
The game you intend to use both BodySlide and Outfit Studio for. Changing this option will adjust the game data path and reference skeleton as well, make sure they're correct.</p>
<p><strong>Game Data Path</strong><br/>
This path is supposed to point to the "Data" folder of the active target game. It is used for telling the program where to load resources from and build the output files of projects to.</p>
<h3>
<a aria-hidden="true" class="anchor" href="#general" id="user-content-general"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>General</h3>
<p><strong>Override Warning</strong><br/>
Enables or disables the selection dialog that appears when multiple projects in a batch build are overriding the same output files.</p>
<p><strong>Left Mouse Pan</strong><br/>
Enables or disables panning with the left mouse button in Outfit Studio. Bear in mind that the left mouse button is also used for brush actions.</p>
<p><strong>BSA Textures</strong><br/>
Enables or disables scanning BSA/BA2 archives in the data folder for resources, such as textures for the preview windows.</p>
<p><strong>Language</strong><br/>
This enables the selected language for the program, as long as the translation files for it exist and are translated.</p>

<p>A selection of all BSA/BA2 archives that are scanned for resources as needed, such as material or texture files. Try to keep these to a minimum for performance reasons.</p>
<h3>
<a aria-hidden="true" class="anchor" href="#reference-skeleton" id="user-content-reference-skeleton"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>Reference Skeleton</h3>
<p><strong>File</strong><br/>
This is the reference skeleton file that Outfit Studio uses to create a list of bones and read bone transforms from.</p>
<p><strong>Root Node</strong><br/>
The root node name can differ from game to game. It needs to be correctly set up in order for Outfit Studio to know where to start reading the list of bones from.</p>
<h2>
<a aria-hidden="true" class="anchor" href="#configxml" id="user-content-configxml"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>Config.xml</h2>
<p>A few options are currently only found in the Config.xml file directly.</p>
<p><strong>GroupAliases</strong><br/>
Group aliases provide a way for presets saved under older group names to still apply when a group has been renamed. You can also merge several groups into one by linking them to eachother.</p>
<p><strong>SliderMinimum and SliderMaximum</strong><br/>
Sets the minimum and maximum slider values for the BodySlide UI. This allows you to experiment more easily with sliders. There might be clipping outside of the 0-100 range and it's not supported!</p>
