          <h1>
<a aria-hidden="true" class="anchor" href="#overview" id="user-content-overview"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>Overview</h1>
<p>The jsonForms library is a small JavaScript client-side library that takes
a set of options and a structured JSON data model as input and returns a
<a href="http://getbootstrap.com/" rel="nofollow">Bootstrap</a>-friendly HTML form that matches the schema.</p>
<p>The generated form includes client-side validation logic using
<a href="http://jqueryvalidation.org/" rel="nofollow">jQuery Validate</a> that provides instant feedback
to the user upon submission. If all the field values are valid, then the form
is submitted and a customizable callback is issued.</p>
<p>This is the reference documentation of the jsonForms Library. If you would rather see the Library in action, check the <a href="https://rawgit.com/jonmash/jsonForms/master/examples/sandbox/sandbox.html" rel="nofollow">jsonForms Sandbox</a> that lets you try and extend the examples of this doc.</p>

<h1>
<a aria-hidden="true" class="anchor" href="#outline-of-jsonforms-objects" id="user-content-outline-of-jsonforms-objects"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a><a name="user-content-outline"></a>Outline of jsonForms objects</h1>
<p>The <code>jsonForms.BuildForm()</code> function takes two javascript objects as parameters. The first object is the options object:</p>
<div class="highlight highlight-source-js"><pre><span class="pl-k">var</span> options <span class="pl-k">=</span> {
    <span class="pl-c"><span class="pl-c">//</span> Object that describes the options sued to configure jsonForms</span>
}</pre></div>
<p>The <code>options</code> property is <em>optional</em>. It sets some options that jsonForms will use to generate the form. If no options are given, the defaults will be used.</p>
<p>The second object is the schema object</p>
<div class="highlight highlight-source-js"><pre><span class="pl-k">var</span> schema <span class="pl-k">=</span> {
    <span class="pl-c"><span class="pl-c">//</span> Array of objects that describes the desired form output</span>
}</pre></div>
<p>The <code>schema</code> object is <em>required</em>. It describes the structure of the form before it is generated. The schema lets you describe the form structure that you'd like to be generated. Check <a href="#schema">Using Schema to describe your data model</a> below for more info.</p>
<p>The <code>submitCallback</code> callback is <em>optional</em> but chances are you will want to do something with the values entered by the user when the form is submitted and that callback is the easiest way to get it. It receives the object generated from the submitted values and the schema, even if the fields are not valid.</p>

<h2>
<a aria-hidden="true" class="anchor" href="#supported-types" id="user-content-supported-types"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a><a name="user-content-schema-supported"></a>Supported types</h2>
<p>jsonForms supports the major HTML form field types:</p>
<ul>
<li>
<code>text</code> for generic strings</li>
<li>
<code>email</code> for strings that match an email</li>
<li>
<code>password</code> for strings that are obfuscated.</li>
<li>
<code>url</code> for strings that match a standard URL</li>
<li>
<code>dropdown</code> for for a dropdown multiple choice option</li>
<li>
<code>checkboxlist</code> for a list of checkboxes where more than one option can be selected</li>
<li>
<code>radiolist</code> for a list of radio buttons where only one option can be selected</li>
<li>
<code>hidden</code> for strings that are hidden to the user.  Useful for <code>ids</code> and <code>keys</code>
</li>
</ul>
<p>jsonForms supports the major HTML button types:</p>
<ul>
<li>
<code>submit</code> generates a <em>submit</em> button. Clicking it will submit the form</li>
<li>
<code>reset</code> generates a <em>reset</em> button. Clicking it will reset the form</li>
</ul>
<p>Special cases:</p>
<ul>
<li>
<code>fieldset</code> is a fieldset that can contain one or more other HTML types.</li>
</ul>
<p>Types are the most straightforward constraint you can put on property values. All properties must have a <code>type</code>, otherwise jsonForms will not know what to do with it. You may add more properties. We've already seen <code>required</code> to enforce the presence of a value and <code>label</code> as a way to set the displayed text next to the field. Note that not all <code>types</code> have the label option.</p>

<a aria-hidden="true" class="anchor" href="#skip-fields-title-the-notitle-property" id="user-content-skip-fields-title-the-notitle-property"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a><a name="user-content-fields-common-notitle"></a>Skip field's title: the <code>notitle</code> property</h3>
<p>Sometimes, you may want JSON Form not to insert a label for the input field. Use this property to ignore the <code>title</code> property of the schema element.</p>
<h3>
<a aria-hidden="true" class="anchor" href="#prepend-a-message-to-a-field-the-prepend-property" id="user-content-prepend-a-message-to-a-field-the-prepend-property"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a><a name="user-content-fields-common-prepend"></a>Prepend a message to a field: the <code>prepend</code> property</h3>
<p>Use this property to insert a prefix before the generated form input. The property accepts an HTML fragment string as value. Note that the prefix will not appear in the final value when the form is submitted.</p>
<p>See also <a href="#fields-common-append"><code>append</code></a></p>
<h3>
<a aria-hidden="true" class="anchor" href="#fields-title-the-title-property" id="user-content-fields-title-the-title-property"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a><a name="user-content-fields-common-title"></a>Field's title: the <code>title</code> property</h3>
<p>By default, JSON Form uses the <code>title</code> property defined in the schema as label for the generated input field. Set this property in the <code>form</code> section to override that of the schema element (or to define it if the schema does not).</p>
<h3>
<a aria-hidden="true" class="anchor" href="#fields-placeholder-the-placeholder-property" id="user-content-fields-placeholder-the-placeholder-property"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a><a name="user-content-fields-common-placeholder"></a>Field's placeholder: the <code>placeholder</code> property</h3>
<p>JSON Form sets the <code>placeholder</code> attribute of the input field based on the value of this property if available.</p>
<h3>
<a aria-hidden="true" class="anchor" href="#disable-a-field-the-disabled-property" id="user-content-disable-a-field-the-disabled-property"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a><a name="user-content-fields-common-disabled"></a>Disable a field: the <code>disabled</code> property</h3>
<p>When set, JSON Form sets the <code>disabled</code> attribute of the underlying input field. Note the <code>disabled</code> state is a purely visual effect: validation does not enforce that the field's value has not changed.</p>
<p>JSON Form also adds a <code>jsonform-disabled</code> class on the container <code>&lt;div&gt;</code> of the generated field to ease styling, if needed.</p>
<h3>
<a aria-hidden="true" class="anchor" href="#mark-a-field-as-read-only-the-readonly-property" id="user-content-mark-a-field-as-read-only-the-readonly-property"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a><a name="user-content-fields-common-readonly"></a>Mark a field as read-only: the <code>readonly</code> property</h3>
<p>When set, JSON Form sets the <code>readonly</code> attribute of the underlying input field, effectively making the field read-only. The property is also purely visual: validation does not enforce that the field's value has not changed.</p>
<p>JSON Form also adds a <code>jsonform-readonly</code> class on the container <code>&lt;div&gt;</code> of the generated field to ease styling, if needed.</p>
<h3>
<a aria-hidden="true" class="anchor" href="#mark-a-field-allow-empty-value-in-the-final-values-the-allowempty-property" id="user-content-mark-a-field-allow-empty-value-in-the-final-values-the-allowempty-property"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a><a name="user-content-fields-common-allowEmpty"></a>Mark a field allow empty value in the final values: the <code>allowEmpty</code> property</h3>
<p>When set, JSON Form will include the field with the empty string value in the final values, else the fields with empty value will not be included in the final values object.</p>

<a aria-hidden="true" class="anchor" href="#selection-fields" id="user-content-selection-fields"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a><a name="user-content-fields-selection"></a>Selection fields</h2>
<p>Selection fields are linked to <code>string</code> properties in the schema that define an <code>enum</code> property with an enumeration of possible values. Such fields allow the selection of one or more options depending on the type of field.</p>
<p>The enumeration defined in the schema sets the list of values that your application will receive once the form is submitted. If you want to replace these values in the form by more user-friendly messages, define the mapping in the <code>titleMap</code> property.</p>
<div class="highlight highlight-source-json"><pre>{
  <span class="pl-s"><span class="pl-pds">"</span>schema<span class="pl-pds">"</span></span>: {
    <span class="pl-s"><span class="pl-pds">"</span>gender<span class="pl-pds">"</span></span>: {
      <span class="pl-s"><span class="pl-pds">"</span>type<span class="pl-pds">"</span></span>: <span class="pl-s"><span class="pl-pds">"</span>string<span class="pl-pds">"</span></span>,
      <span class="pl-s"><span class="pl-pds">"</span>title<span class="pl-pds">"</span></span>: <span class="pl-s"><span class="pl-pds">"</span>Gender<span class="pl-pds">"</span></span>,
      <span class="pl-s"><span class="pl-pds">"</span>enum<span class="pl-pds">"</span></span>: [ <span class="pl-s"><span class="pl-pds">"</span>male<span class="pl-pds">"</span></span>, <span class="pl-s"><span class="pl-pds">"</span>female<span class="pl-pds">"</span></span>, <span class="pl-s"><span class="pl-pds">"</span>alien<span class="pl-pds">"</span></span> ]
    }
  },
  <span class="pl-s"><span class="pl-pds">"</span>form<span class="pl-pds">"</span></span>: [
    {
      <span class="pl-s"><span class="pl-pds">"</span>key<span class="pl-pds">"</span></span>: <span class="pl-s"><span class="pl-pds">"</span>gender<span class="pl-pds">"</span></span>,
      <span class="pl-s"><span class="pl-pds">"</span>titleMap<span class="pl-pds">"</span></span>: {
        <span class="pl-s"><span class="pl-pds">"</span>male<span class="pl-pds">"</span></span>: <span class="pl-s"><span class="pl-pds">"</span>Dude<span class="pl-pds">"</span></span>,
        <span class="pl-s"><span class="pl-pds">"</span>female<span class="pl-pds">"</span></span>: <span class="pl-s"><span class="pl-pds">"</span>Dudette<span class="pl-pds">"</span></span>,
        <span class="pl-s"><span class="pl-pds">"</span>alien<span class="pl-pds">"</span></span>: <span class="pl-s"><span class="pl-pds">"</span>I'm from outer space!<span class="pl-pds">"</span></span>
      }
    }
  ]
}</pre></div>

<h2>
<a aria-hidden="true" class="anchor" href="#groups-of-fields" id="user-content-groups-of-fields"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a><a name="user-content-fields-groups"></a>Groups of fields</h2>
<p>It is often convenient to group a set of fields together in a form to ease readability and thus improve the user experience. JSON Form provides a few types for that purpose.</p>
<h3>
<a aria-hidden="true" class="anchor" href="#set-of-fields-the-fieldset-type" id="user-content-set-of-fields-the-fieldset-type"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a><a name="user-content-fields-fieldset"></a>Set of fields: the <code>fieldset</code> type</h3>
<p>JSON Form generates a <code>fieldset</code> for all schema properties of type <code>object</code>, but you may also want to generate a fieldset without referencing a schema property at all. As you might expect from the name, a <code>fieldset</code> field simply produces a <code>&lt;fieldset&gt;</code> element in the resulting form.</p>
<p>When set, the <code>expandable</code> boolean property tells JSON Form to render the fieldset's title and only expand the fieldset's section when the user clicks on it. This is typically useful for advanced options that users could skip on first read.</p>
<p>If you're using Bootstrap to style the resulting form, please note that it does not include styles for <em>expandable</em> sections at first. You might want to add a few extra stylesheet properties to your CSS to add a visual hint that the section can be expanded/collapsed, for instance:</p>
<div class="highlight highlight-source-css"><pre><span class="pl-e">.expandable</span> <span class="pl-k">&gt;</span> <span class="pl-ent">legend</span><span class="pl-e">:before</span> {
  <span class="pl-c1"><span class="pl-c1">content</span></span>: <span class="pl-s"><span class="pl-pds">'</span><span class="pl-cce">\25B8</span><span class="pl-pds">'</span></span>;
  <span class="pl-c1"><span class="pl-c1">padding-right</span></span>: <span class="pl-c1">5<span class="pl-k">px</span></span>;
}
<span class="pl-e">.expanded</span> <span class="pl-k">&gt;</span> <span class="pl-ent">legend</span><span class="pl-e">:before</span> {
  <span class="pl-c1"><span class="pl-c1">content</span></span>: <span class="pl-s"><span class="pl-pds">'</span><span class="pl-cce">\25BE</span><span class="pl-pds">'</span></span>;
}</pre></div>

      <span class="pl-s"><span class="pl-pds">"</span>type<span class="pl-pds">"</span></span>: <span class="pl-s"><span class="pl-pds">"</span>string<span class="pl-pds">"</span></span>,
      <span class="pl-s"><span class="pl-pds">"</span>title<span class="pl-pds">"</span></span>: <span class="pl-s"><span class="pl-pds">"</span>Comment<span class="pl-pds">"</span></span>
    },
    <span class="pl-s"><span class="pl-pds">"</span>name<span class="pl-pds">"</span></span>: {
      <span class="pl-s"><span class="pl-pds">"</span>type<span class="pl-pds">"</span></span>: <span class="pl-s"><span class="pl-pds">"</span>string<span class="pl-pds">"</span></span>,
      <span class="pl-s"><span class="pl-pds">"</span>title<span class="pl-pds">"</span></span>: <span class="pl-s"><span class="pl-pds">"</span>Name<span class="pl-pds">"</span></span>
    },
    <span class="pl-s"><span class="pl-pds">"</span>age<span class="pl-pds">"</span></span>: {
      <span class="pl-s"><span class="pl-pds">"</span>type<span class="pl-pds">"</span></span>: <span class="pl-s"><span class="pl-pds">"</span>number<span class="pl-pds">"</span></span>,
      <span class="pl-s"><span class="pl-pds">"</span>title<span class="pl-pds">"</span></span>: <span class="pl-s"><span class="pl-pds">"</span>Age<span class="pl-pds">"</span></span>
    }
  },
  <span class="pl-s"><span class="pl-pds">"</span>form<span class="pl-pds">"</span></span>: [
    {
      <span class="pl-s"><span class="pl-pds">"</span>key<span class="pl-pds">"</span></span>: <span class="pl-s"><span class="pl-pds">"</span>comment<span class="pl-pds">"</span></span>,

<h3>
<a aria-hidden="true" class="anchor" href="#submit-the-form-the-submit-type" id="user-content-submit-the-form-the-submit-type"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a><a name="user-content-fields-submit"></a>Submit the form: the <code>submit</code> type</h3>

<h1>
<a aria-hidden="true" class="anchor" href="#form-submission" id="user-content-form-submission"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a><a name="user-content-submission"></a>Form submission</h1>
<p>The form gets submitted when the user clicks the <em>submit</em> button (so don't forget to include one... see <a href="#fields-submit">Submit the form: the <code>submit</code> type</a>). Upon submission, JSON Form:</p>
<ol>
<li>retrieves all fields values</li>
<li>creates the result object from submitted values based on the schema</li>
<li>runs the validator if it is available to check constraints</li>
<li>highlights potential errors in the form next to the problematic fields</li>
<li>reports potential errors and/or passes the result object to the caller</li>
</ol>
<p>Steps 3 and 4 are optional.</p>
<h2>
<a aria-hidden="true" class="anchor" href="#validation" id="user-content-validation"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a><a name="user-content-submission-validation"></a>Validation</h2>
<p>Validation of the values entered by the user relies on the <a href="https://github.com/garycourt/JSV">JSON Schema Validator</a> library which must be loaded if validation is to be enabled.</p>
<p>Validation is enabled by default. To disable validation, set the <code>validate</code> property of the JSON Form object to <code>false</code>, as in:</p>
<div class="highlight highlight-source-json"><pre>{
  <span class="pl-s"><span class="pl-pds">"</span>schema<span class="pl-pds">"</span></span>: {<span class="pl-ii">...</span>},
  <span class="pl-s"><span class="pl-pds">"</span>form<span class="pl-pds">"</span></span>: [<span class="pl-ii">...</span>],
  <span class="pl-s"><span class="pl-pds">"</span>validate<span class="pl-pds">"</span></span>: <span class="pl-c1">false</span>
}</pre></div>
<h2>
<a aria-hidden="true" class="anchor" href="#error-reporting" id="user-content-error-reporting"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a><a name="user-content-submission-error"></a>Error reporting</h2>
<p>JSON Form highlights errors automatically next to problematic fields, with an error message that tells user why the entered values are wrong.</p>
<h2>
<a aria-hidden="true" class="anchor" href="#accessing-submitted-values" id="user-content-accessing-submitted-values"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a><a name="user-content-submission-values"></a>Accessing submitted values</h2>
<p>JSON Form calls the <code>onSubmitValid</code> callback function with the result object generated from the submitted values when the form is submitted and when the validation did not report any error.</p>
<div class="highlight highlight-source-js"><pre><span class="pl-k">var</span> formObject <span class="pl-k">=</span> {
  <span class="pl-s"><span class="pl-pds">"</span>schema<span class="pl-pds">"</span></span><span class="pl-k">:</span> {<span class="pl-k">...</span>},
  <span class="pl-s"><span class="pl-pds">"</span>form<span class="pl-pds">"</span></span><span class="pl-k">:</span> [<span class="pl-k">...</span>],
  <span class="pl-s"><span class="pl-pds">"</span><span class="pl-en">onSubmitValid</span><span class="pl-pds">"</span></span><span class="pl-k">:</span> <span class="pl-k">function</span> (<span class="pl-smi">values</span>) {
    <span class="pl-c"><span class="pl-c">//</span> "values" follows the schema, yeepee!</span>
    <span class="pl-en">console</span>.<span class="pl-c1">log</span>(values);
  }
};</pre></div>
<p>If you would like to report a global error message to the user when submitted values are incorrect, use the <code>onSubmit</code> function that gets called no matter what once the validation is over.</p>
<div class="highlight highlight-source-js"><pre><span class="pl-k">var</span> formObject <span class="pl-k">=</span> {
  <span class="pl-s"><span class="pl-pds">"</span>schema<span class="pl-pds">"</span></span><span class="pl-k">:</span> {<span class="pl-k">...</span>},
  <span class="pl-s"><span class="pl-pds">"</span>form<span class="pl-pds">"</span></span><span class="pl-k">:</span> [<span class="pl-k">...</span>],
  <span class="pl-s"><span class="pl-pds">"</span><span class="pl-en">onSubmit</span><span class="pl-pds">"</span></span><span class="pl-k">:</span> <span class="pl-k">function</span> (<span class="pl-smi">errors</span>, <span class="pl-smi">values</span>) {
    <span class="pl-k">if</span> (errors) {
      <span class="pl-en">alert</span>(<span class="pl-s"><span class="pl-pds">'</span>Check the form for invalid values!<span class="pl-pds">'</span></span>);
      <span class="pl-k">return</span>;
    }
    <span class="pl-c"><span class="pl-c">//</span> "values" follows the schema, yeepee!</span>
    <span class="pl-en">console</span>.<span class="pl-c1">log</span>(values);
  }
};</pre></div>

<h1>
<a aria-hidden="true" class="anchor" href="#adding-styles-to-the-generated-form" id="user-content-adding-styles-to-the-generated-form"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a><a name="user-content-styles"></a>Adding styles to the generated form</h1>
<p>The JSON Form library generates HTML and a bit of JavaScript to enhance the form, but it does not generate any single line of CSS. That's on purpose! The markup produced by JSON Form is straightforward and <a href="http://twitter.github.com/bootstrap/">Bootstrap</a>-friendly. This means that you may simply include the Bootstrap CSS file to create a visually appealing form.</p>
<p>Relying on the Bootstrap CSS is by no means compulsory. You may extend it with additional properties or use your own stylesheets. If you need to flag certain tags to ease CSS authoring, use the <code>htmlClass</code> property (see <a href="#wiki-fields-common-htmlclass">Style the field container: the htmlClass property</a>.</p>

<a aria-hidden="true" class="anchor" href="#dependencies" id="user-content-dependencies"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a><a name="user-content-deps"></a>Dependencies</h1>
<p>The jsonForms library requires:</p>
<ul>
<li><a href="http://jquery.com/" rel="nofollow">jQuery</a></li>
<li>The <a href="http://jqueryvalidation.org/" rel="nofollow">jQuery Validate</a> plugin</li>
</ul>
<p>The jsonForms library may require further libraries, depending on the features you need for the forms you need to render. In particular:</p>
<ul>
<li>
<a href="http://getbootstrap.com/" rel="nofollow">Bootstrap CSS</a> v3.3.1 or above is more or less needed. You can get away without it but all of the jsonForms forms are rendered with Bootstrap in mind.</li>
</ul>
<p><img alt="alt tag" src="https://raw.githubusercontent.com/jonmash/jsonForms/master/examples/compare.png"/></p>
<p>All of these libraries are in the repository, although you might want to check their respective Web site for more recent versions.</p>
<p>Note: jsonForms uses <code>JSON.parse</code> and <code>JSON.stringify</code> which are normally supported by modern browsers. If you need to support other browsers, consider adding a JSON library.</p>
